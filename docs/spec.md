# NewTabPad 仕様

ユーザから見た振る舞いを定義する。技術的な実装方針は `architecture.md` を参照。

## 概要

Chrome の新しいタブを、シンプルなプレーンテキストのメモ帳に置き換える拡張機能。
複数のメモを保持でき、タブを跨いで内容が同期される。

## 動作概要

### 新しいタブを開いたとき

- 既存のメモが1件以上ある場合: 最後に編集したメモ（`updatedAt` が最新のもの）を開く
- メモが1件もない場合: 空のエディタを表示する
- 起動直後に textarea へ自動フォーカスはしない（アドレスバーで検索可能な状態を維持するため）

### メモの作成

- 新規メモ作成は明示的な操作（「新規」ボタン または ショートカット）で行う
- 新規エディタを開いた直後はストレージに何も保存されていない
- ユーザが最初の文字を入力した瞬間にメモがストレージへ作成され、その時刻が `createdAt` となる
- 入力のたびに `updatedAt` が更新される（保存自体は debounce される）

### 自動保存

- 入力停止から 500ms 後に保存（debounce）
- IME 変換確定（`compositionend`）に連動した即保存は行わない
- タブを閉じる直前の `beforeunload` 強制保存も行わない（debounce のタイムウィンドウを信頼する）

### 空メモの扱い

- 「空メモ」は `content.trim() === ""` を満たすメモ
- 以下のタイミングで空メモを破棄する:
  1. 自動保存の発火時に空であれば、書き込まずに削除（既存なら削除、未作成なら何もしない）
  2. 「新規」または「一覧から別メモを開く」で現在のメモから離れる際
  3. アプリ起動時のスキャン（防御的クリーンアップ）

### メモの切替

- 「新規」ボタン: 現在のメモを保存（空なら破棄ルール）→ 空のエディタを表示
- 一覧から別メモを開く: 現在のメモを保存 → 選択したメモを開く

## エディタ画面

### 見た目

- 配色: 落ち着いた、低彩度のテーマ（白系／淡いグレー基調）
- フォント: システム等幅フォントを優先順に指定
  - macOS: SF Mono, Menlo, Monaco
  - Windows: Consolas, "Courier New"
  - 共通フォールバック: monospace
- 行番号: 左側に固定幅で表示（最大 5 桁 = 99999 行を想定した幅）
- 行折り返し: なし。長い行は横スクロール

### 行番号

- エディタ本体と縦スクロールが同期する
- クリックしても何も起きない（テキスト選択にも混入しない）

### 入力

- 普通のテキストエディタ相当の挙動
- IME 入力中（`compositionstart` から `compositionend` まで）は他タブからの更新を保留する（後述）

## 一覧画面

### 表示項目

各メモについて以下を行で表示:

- 作成日時 `YYYY-MM-DD HH:MM`（ローカルタイムゾーン、絶対時刻）
- プレビュー: メモ内容の最初の非空行を 30 文字で切り詰め（超過時は末尾 `…`）
- 削除ボタン

### 並び順

- `updatedAt` 降順
- 同値の場合は `createdAt` 降順をタイブレーク

### 操作

- 行をクリックでそのメモをエディタで開く（その時点で現在編集中のメモは保存される）
- 削除ボタンでそのメモを削除（ネイティブの `confirm()` で確認、削除後の取り消しはなし）
- `Esc` キー、または ショートカット で一覧表示を閉じてエディタへ戻る

## キーボードショートカット

| 操作 | macOS | Windows / Linux |
|---|---|---|
| 新規メモ作成 | `Cmd + Shift + Enter` | `Ctrl + Shift + Enter` |
| メモ一覧表示 | `Cmd + Shift + L` | `Ctrl + Shift + L` |
| 一覧からエディタへ戻る | `Esc` | `Esc` |

- いずれもエディタにフォーカスがある状態でも発火する
- Chrome 標準ショートカットとは衝突しないことを確認済み

## タブ間同期（ユーザから見た挙動）

- 同じメモを複数タブで開いている場合、片方の編集が他方に反映される
- 反映時、カーソル位置の保全はしない（内容は単純に上書きされる）
- IME 入力中は反映が保留され、変換確定後にまとめて反映される
- 競合検出は行わない。同時編集が起きた場合は最後に保存された内容が残る
- 別タブでメモを削除した場合:
  - 自分が開いていたメモなら、最新の `updatedAt` のメモへフォールバック
  - 残るメモがなければ空のエディタへ
- 一覧表示中に別タブで作成／削除／編集が起きた場合、一覧は自動再描画される

## ストレージ（ユーザ視点）

- 全データはローカル（`chrome.storage.local`）に保存される
- 件数の上限はない
- データはこの拡張機能専用。ブラウザ間同期、エクスポート／インポートは v1 では持たない

## v1 の対象外

- 検索、タグ、カテゴリ
- インポート／エクスポート
- リッチテキスト編集、マークダウンプレビュー
- リモート同期サービス連携
- 複数行の折り返し表示
- ユーザ設定（フォント、テーマ、debounce 時間など）

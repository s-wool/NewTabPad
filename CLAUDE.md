# CLAUDE.md — NewTabPad

このファイルは Claude Code がこのリポジトリで作業する際のコンテキスト集です。

## プロジェクト概要

**NewTabPad** は、Chrome の新しいタブをプレーンテキストのメモ帳に置き換える Chrome 拡張機能です。

- Manifest V3
- ビルドプロセスなし（素の HTML / CSS / JS、ES モジュール）
- ストレージは `chrome.storage.local`（`unlimitedStorage` permission）
- 単一ユーザの個人利用、複数タブ間で同期

## 開発フロー

ビルドステップなし。Chrome の `chrome://extensions` で「パッケージ化されていない拡張機能を読み込む」でリポジトリのルートを指定する。

変更を反映するには:

1. ファイルを編集
2. `chrome://extensions` でこの拡張のリロードボタンをクリック
3. 新しいタブを開いて確認

自動テストは v1 では持たない。手動の確認観点:

- 新規メモの作成 → 文字入力で `createdAt` が記録される
- debounce 500ms で保存される
- タブを閉じて開き直すと最新メモが復元される
- 複数タブで同じメモを開いて編集が同期する
- IME 入力中に他タブが編集しても変換が壊れない
- 一覧表示中に他タブで作成・削除された変化が一覧に反映される

## 設計上の重要な制約

実装中に「これも入れた方がいいのでは？」と思っても、以下は v1 ではやらない（仕様で明示的に除外している）:

- リッチテキスト、マークダウンプレビュー
- 検索／タグ／カテゴリ
- リモート同期、エクスポート／インポート
- 競合検出、差分マージ
- `beforeunload` 強制保存（debounce 500ms を信頼する）
- カスタム UI の確認ダイアログ（ネイティブ `confirm()` を使う）
- 新規タブ起動時の textarea 自動フォーカス（**ユーザが意図的にこれを除外している。アドレスバー検索を妨げないため**）

これらを追加する判断はユーザの明示的な合意が必要。

## 実装上の落とし穴（要注意ポイント）

### IME（日本語入力）

- `<textarea>` の `compositionstart` と `compositionend` を必ず購読する
- IME 中（`isComposing === true`）は他タブからの更新を `pendingValue` に保留し、`compositionend` で適用する
- これがないと変換途中で `textarea.value` が書き換わって日本語入力が壊れる

### タブ間同期の sourceTabId

- 各タブは `crypto.randomUUID()` で生成した `myTabId` をメモリに保持（ストレージには永続化しない）
- 書き込み時は `__sync.sourceTabId = myTabId` を一緒にセット
- `chrome.storage.onChanged` で受信時、`__sync.sourceTabId === myTabId` なら自分発なので無視
- これがないと自分の保存ループで自分のエディタが書き換わるケースが起こる

### 行番号と折り返し

- `<textarea wrap="off">` で折り返し無効化（論理行 = 表示行を保証）
- 行折り返しを許すと行番号がずれるので**入れてはいけない**
- gutter は textarea の `scroll` イベントで `scrollTop` を同期する

### 空メモの破棄

- `content.trim() === ""` を空とみなす
- 保存時、メモ切替時、起動時の3箇所でクリーンアップする
- どれか1つでも忘れると「空メモが残骸として残る」状態になる

## キーボードショートカット

| 操作 | macOS | Windows / Linux |
|---|---|---|
| 新規メモ作成 | `Cmd + Shift + Enter` | `Ctrl + Shift + Enter` |
| メモ一覧表示 | `Cmd + Shift + L` | `Ctrl + Shift + L` |
| 一覧からエディタへ戻る | `Esc` | `Esc` |

これらは Chrome 標準ショートカットと衝突しないよう選定済み。**変更する際は同じレベルの検討（Chrome 標準との衝突確認、AltGr 衝突懸念など）が必要**。
